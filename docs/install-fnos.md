# 飞牛 fnOS 安装指南（原生应用，推荐）

原生 .fpk 应用：应用中心一键安装、桌面有图标、升级保数据，**不需要 Docker**。

## 要求

- 飞牛 fnOS（x86_64 机型）
- 应用中心已安装 **Python 3.12**（`python312`；未装的话安装 NAS Photo 时会提示）

## 安装步骤

1. 到 [Releases](../../../releases) 下载最新的 `nas-photo_<版本>_x86.fpk`
2. 把文件传到飞牛（网页版文件管理上传，或 SMB 拷入任意目录）
3. **应用中心 → 右上角 → 手动安装**，选择这个 .fpk 文件
   - 若提示缺少依赖 `python312`，先在应用中心装它，再重新安装
4. 安装完成后**启用**应用 → 桌面出现「NAS Photo」图标
5. 点图标打开 Web 界面 → **注册新账号**（第一个账号即管理员）

> 也可以直接用浏览器访问 `http://<飞牛IP>:9277`。

## 授权照片文件夹

原生应用只能访问你明确授权的目录：

1. 飞牛 **设置 → 应用 → NAS Photo → 文件夹权限**，添加你的照片目录
2. 回到 NAS Photo：**设置 → 图库管理 → 添加图库**，路径浏览器里选择刚授权的目录
3. 保存 → **扫描**，完成后时间线出图

> ⚠️ 应用内「永久删除」会真实删除原文件。第一次建议先授权一个测试子目录，确认符合预期再接入整库。

## 开启 AI（可选）

**设置 → AI** → 一键下载（约 3.4GB，国内自动走镜像源，10–20 分钟视网速）。
下载到应用数据目录，升级不用重下。完成后即可使用：人脸聚类、自然语言搜图、文字搜索。

## 升级

下载新版 .fpk → 应用中心手动安装 → 覆盖安装即完成，账号、索引、AI 模型全部保留。

## 常见问题

| 现象 | 处理 |
|---|---|
| 安装时提示缺依赖 | 应用中心先装 Python 3.12（python312） |
| 启用失败 / 提示端口被占用 | 先禁用再启用一次；仍不行请到 Issues 反馈并附机型与 fnOS 版本 |
| 图库里看不到照片目录 | 检查「文件夹权限」是否已授权该目录 |
| 打不开界面 | 确认应用已启用；防火墙放行 9277 端口 |
